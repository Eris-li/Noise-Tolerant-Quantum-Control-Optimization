# Security Policy

This project is scientific research software. It does not intentionally provide
network services, authentication, or data-processing pipelines for sensitive
personal information.

## Reporting a Vulnerability

Please report security issues privately through the repository owner contact on
GitHub, or by opening a GitHub security advisory if the repository settings
permit it.

When reporting, include:

- the affected commit or release;
- a minimal reproduction;
- the expected impact;
- whether the issue affects local execution only or published artifacts.

Do not include private credentials, unpublished experimental data, or personal
information in public issues.

## Supported Versions

The `main` branch is the only supported development line until the project
starts publishing tagged releases.
